## **What is a Frontend Developer?**

> A frontend developer is a professional who uses HTML, CSS, and JavaScript to design
> and build the visual and interactive elements of websites and applications that users engage with directly.

frontend 開発者は ～というプロフェッショナルです
HTML, CSS, JavaScript でデザインしたり、見た目を構築したり、ユーザーが直接かかわるウェブサイトやアプリの
要素を相互作用させるものである

> They ensure the interface is responsive, accessible, and visually appealing.
> Every feature you see and interact with on a website (like buttons, menus, and animations) is created by a frontend developer.

彼らはインタフェースをレスポンシブに、アクセスしやすく、そして見た目のアピールさせます
ウェブサイトにあなたが見る、そして触っている要素すべては frontend 開発者がすべて作っています

## **Which languages are used for Frontend Development?**

> The easy part about the frontend development role is that there aren’t that many options to choose
> from when it comes to [**frontend languages**](https://roadmap.sh/frontend/languages) and [**technologies**](https://roadmap.sh/frontend/technologies) (unlike with backend development).

Frontend開発者の役割について簡単な部分は、
バックエンドの開発とは違ってFrontendで使われる言語と技術について関して言えば
そこまで多くの選択肢があるわけではない、ということです
（Backend開発者とは違って）

> As a frontend developer, you’ll be working with the following technologies:
> HTML : The markup language used to create the skeleton of the page.
> All the information you want to show on a webpage will be laid out through HTML.

Frontend開発者にとって以下の技術を扱いことになるでしょう
HTML：マークアップ言語はページの骨格を作るために使われます
ウェブページ上に表示したいすべての情報は
HTMLを通して配置されます

> CSS : The Cascading Style Sheet language allows you to adjust the way in which the HTML elements are rendered,
> improving the visuals of your webpage.

CSS：カスケードスタイルシート言語は、あなたに～その方法を調整することを可能にするでしょう

～HTMLの要素をレンダーして、ウェブページの見た目を改善する

> JavaScript : This is the de facto programming language for frontend development, and it allows you to add dynamism to your websites/web apps.
> There is an alternative known as TypeScript, which is a strongly typed superset of JavaScript that you can use instead.

JavaScript：Frontend開発者とってデファクトスタンダードになっている言語です
ウェブサイトやウェブアプリに動的な要素を追加することを可能にします
そして代替として知られるTypeScriptという言語は、JavaScriptの代わりに使える言語として、
JavaScriptの強力な型を持っているスーパーセットです

> However, in that scenario, you’d have to set up a transpiler to translate your code into JavaScript before being able to run it in the browser.

しかしながら、このシナリオでは（TypeScriptを利用する）ブラウザで実行可能にする前に
TypeScriptをJavaScriptに変換するTranspilerをセットアップする必要があります

## **What skills are required for a Frontend developer?**

> The main skills required to become a frontend developer are HTML, CSS, and JavaScript.
> The rest are also important, but without those three basic ones, you can’t apply any of the others.
> **The full list** of [**frontend developer skills**](https://roadmap.sh/frontend/developer-skills) you should look into if you’re hoping to up your game **is the following**:

Frontend開発者になるための主要な技術力はHTML, CSS, JavaScript です
残りの技術力も当然重要ですが、この基本3種の技術力を持っていなければ
ほかのすべての物に応用することはできません
技術力を上げたいと願っているのであれば、あなたが見るべきFrontend開発者全集は以下の通りです

> 1. Understanding the core technologies: HTML, CSS & JavaScript.
> 2. Understanding accessibility: Knowing how it can affect your users' experience.
> 3. Version control systems: Be familiar with tools like Git.

1. HTML, CSS, Javascript コアになる技術を理解すること
2. アクセシビリティへの理解　ユーザー体験にどのように影響をするのか
3. Gitのようなバージョンコントロールシステムに習熟すること

> 4. Responsive design: Implementing designs that adapt to different devices and screen sizes.
> 5. User experience (UX): Understanding the basics of UX principles.
> 6. SEO: Knowing how search engine optimization works and how you can leverage it in your code.

1. レスポンシブデザイン：デザインを実装すること、異なるデバイスやスクリーンサイズに適合する
2. ユーザー体験：UXの基本原理の基礎について理解すること
3. SEO：サーチエンジンの最適がどのように動作するか、またコードにどのように活用できるのかを知ること

> 7. RESTful APIs: Understanding the basics of RESTful APIs and how to consume them.
> 8. Testing and debugging: Implementing testing practices and debugging effectively.
> 9. Dev tools: Master the browser’s developer tools for inspecting, debugging, and optimizing code.

1. RESTful API：RESTful API の基礎、そしてそれがどのように使われるのかを理解する
2. テスト、そしてデバッグ：効果的なテスト手法の導入やデバッグを実装すること
3. 開発ツール：ブラウザの開発者ツールの使い方マスター、検証・デバッグ・最適化コードのための

> 10. Frameworks: Having a go-to frontend framework (like React or Vue) and understanding others at a high level.
> 11. Web performance: Understanding web performance optimizations and core web vitals.
> 12. TypeScript: Understanding how TypeScript works and when to use it.

1. Frameworks：ReactやVueなどのお気に入りのFrontendフレームワークやほかのフレームワークのハイレベルな理解を行うこと
2. Webパフォーマンス：Webパフォーマンスの最適化、そして主要なWeb指標を理解すること
3. TypeScript：TypeScriptがどのように動いて、そしていつ使うのかを理解すること

> Covering these basic skills will prepare you for any type of [frontend developer interview questions](https://roadmap.sh/questions/frontend) you might encounter in the future and will enhance your current role.

3つの基本的な技術をカバーすることは、将来受けるであろうあらゆるタイプのFrontend開発者のインタビュー質問に準備することに

なるし、自分の今の役割を強化することになる
